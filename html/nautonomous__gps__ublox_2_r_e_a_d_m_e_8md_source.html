1 # nautonomous gps ublox {#nautonomous_gps_ublox}
2 The ublox package provides support for [u-blox](http://www.u-blox.com) GPS receivers. Only the _serial_ configuration of the driver is documented here, but TCP communication is also supported by the driver (untested).
3 
4 The driver was originally written by Johannes Meyer. Changes made later are detailed in the version history below.
5 
6 ## Options
7 
8 The `ublox_gps` node supports the following parameters:
9 * `device`: Path to the device in `/dev`. Defaults to `/dev/ttyUSB0`.
10 * `baudrate`: Bit rate of the serial communication. Defaults to 9600.
11 * `ublox_version`: Version of device: 6,7 or 8. Defaults to 6. Please consult known issues section.
12 * `frame_id`: ROS name prepended to frames produced by the node. Defaults to `gps`.
13 * `rate`: Rate in Hz of measurements. Defaults to 4.
14 * `enable_sbas`: Enable satellite-based augmentation system. Defaults to false.
15 * `enable_glonass`: Enable GLONASS receiver. Defaults to false.
16 * `enable_beidou`: Enable BeiDou receiver. Defaults to false.
17 * `enable_ppp`: Enable precise-point-positioning system. Defaults to false.
18 * `dynamic_model`: See U-blox protocol spec. Defaults to `portable`.
19 * `fix_mode`: Type of fixes supported: `2d`, `3d` or `both`.
20 * `dr_limit`: Max time in seconds to use dead reckoning after signal is lost. Defaults to 0. (Untested as of 28/08/2014).
21 
22 A sample launch file is provided in `ublox_gps.launch`. The two topics to which you should subscribe are `/ublox_gps/fix` and `/ublox_gps/fix_velocity`. The angular component of `fix_velocity` is unused.

52 ## Known Issues
53 
54 ### Note on `ublox_version`:
55 
56 `ublox_version` is included as a temporary workaround, resulting from the problem that the ROS serializer cannot properly parse _all_ the ublox packets (particularly those with dynamic arrays). It is presently difficult to read the supported features of each device.
57 
58 Consequently, the user should specify the version of their device in rosparam.
59 
60 - Version <= 6: `enable_glonass`, `enable_beidou` are both ignored.
61 - Version == 7: `enable_beidou` is ignored.
62 - Version >= 8: Both options are supported.
63 
64 A warning will be issued on the console indicating which options have been ignored.
